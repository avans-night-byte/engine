{\footnotesize\ttfamily void($\ast$ wl\+\_\+buffer\+\_\+listener\+::release) (void $\ast$data, struct wl\+\_\+buffer $\ast$wl\+\_\+buffer)}

compositor releases buffer

Sent when this wl\+\_\+buffer is no longer used by the compositor. The client is now free to reuse or destroy this buffer and its backing storage.

If a client receives a release event before the frame callback requested in the same wl\+\_\+surface.\+commit that attaches this wl\+\_\+buffer to a surface, then the client is immediately free to reuse the buffer and its backing storage, and does not need a second buffer for the next surface content update. Typically this is possible, when the compositor maintains a copy of the wl\+\_\+surface contents, e.\+g. as a GL texture. This is an important optimization for G\+L(\+E\+S) compositors with wl\+\_\+shm clients.
